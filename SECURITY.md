# Security Policy

## Reporting a Vulnerability

Please report any security concerns privately using GitHub’s “Report a vulnerability” feature in this repository’s Security tab.

**Do not** open public GitHub issues for security-related matters.

When submitting a report, include the following details whenever possible:

- The affected project/repository and version(s)
- The impact and component(s) involved
- Steps to reproduce or a proof of concept (if available)
- Your contact information and preferred name for credit

## Coordination & Disclosure

We adhere to coordinated vulnerability disclosure practices:

- We will acknowledge your report, evaluate the issue, and begin working on a fix.
- We aim to provide periodic updates until the issue is resolved.
